# Отчет о реализации проекта: «Моделирование с использованием эмбеддингов»

## Введение и постановка задачи
Данный проект посвящен созданию системы автоматической группировки (кластеризации) текстов по смыслу и их последующей автоматической разметке. 
Вместо того чтобы человек читал тысячи строк, система сама находит общие темы и называет их.

## Цель проекта 
Разработать программный модуль, который преобразует набор произвольных текстов на русском языке в структурированные тематические группы с автоматическим присвоением названий каждой группе.

## Задачи 
1. Изучить и применить открытые модели-энкодеры от VK (коллекция deepvk).
2. Преобразовать текстовые данные в векторные представления (эмбеддинги).
3. Провести сравнительный анализ алгоритмов кластеризации (K-Means vs HDBSCAN).
4. Визуализировать полученные данные для оценки качества разделения.
5. Интегрировать легковесную языковую модель (LLM) для автоматической генерации названий кластеров. 

**Ожидаемый результат:** Работающий пайплайн (цепочка обработки), который на выходе выдает список категорий с примерами текстов и их кратким описанием.

## Выбор инструментов и данных
Для реализации проекта были использованы материалы из коллекции Sentence Encoders (deepvk), доступной на Hugging Face.

- Модель: `deepvk/USER-bge-m3`. Это мощный мультиязычный энкодер, оптимизированный для русского языка. 
Я использовал его для «оцифровки» смысла предложений — перевода текста в векторы размерностью 1024.

- Датасет: `deepvk/ru-WANLI`. 
Для обучения и тестирования системы был взят фрагмент этого датасета (5000 записей), так как он содержит качественные естественные предложения на русском языке, которые хорошо подходят для проверки кластеризации.

## Реализация и обучение модели
В ходе работы над проектом была выстроена следующая техническая цепочка:
### Этап 1: Получение эмбеддингов
Используя библиотеку sentence_transformers, мы загрузили модель USER-bge-m3 на GPU. Каждое предложение из выборки было преобразовано в вектор. Это позволяет нам вычислять «расстояние» между смыслами предложений математически.

### Этап 2: Поиск оптимального алгоритма
Сначала я попробовал классический метод K-Means. Однако, используя «метод локтя» (Elbow Method) и визуализацию через t-SNE, стало ясно, что для текстовых данных он работает плохо: данные распределены плотно, и четких границ между 4-10 кластерами не наблюдается.
Затем стратегия была изменена:
* UMAP: Использован для снижения размерности с сохранением локальной структуры данных (до 10 компонент).
* HDBSCAN: Этот алгоритм, в отличие от K-Means, умеет находить кластеры произвольной формы и сам определяет их оптимальное количество, помечая слишком специфичные данные как «шум».

### Этап 3: Интерпретация (LLM)
Чтобы кластеры не были просто набором цифр (ID: 1, ID: 2...), была подключена модель Qwen3.5-2B (через библиотеку transformers). Для каждого кластера выбиралась случайная выборка текстов, которая подавалась на вход LLM с промптом: "Проанализируй тексты и придумай одно название для группы".

## Результаты
В ходе реализации проекта удалось добиться следующих результатов:

1. Сочетание UMAP + HDBSCAN показало отличные результаты. На визуализации отчетливо видны сформированные «островки» смыслов. 
2. Система полностью автономна. Она берет «сырой» текст и выдает названия кластеров и примеры предложений, входящих туда.
3. Благодаря модели от VK, система понимает сложные семантические связи (например, понимает, что предложения про «автомобиль» и «транспортное средство» близки по смыслу, даже если у них нет общих слов).
